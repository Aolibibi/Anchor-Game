# 《安科勇者》分工与开发计划 v1.0

> **基于 game_design_v0.9.md · 48小时Game Jam · 多人协同瀑布式开发**

## 0. 小游戏选型（3个，从8个候选中精选）

### 选型标准
- 操作方式互补（移动/画线/点击），覆盖面广
- 实现难度适中，48小时内可完成
- 与安科主题契合，节目效果好

### 最终3个小游戏

| 小游戏 | 操作 | 时长 | 选型理由 |
|--------|------|------|---------|
| **躲避弹幕** | 鼠标移动 | 15-20秒 | 经典Undertale式，紧张刺激，必选招牌 |
| **画线连符文** | 鼠标画线 | 15-20秒 | 有解谜感，适合节点复用，视觉表现力强 |
| **拉霸/老虎机** | 鼠标点击停止 | 10-15秒 | 完美契合安科骰子主题，简单上手，节目效果拉满 |

**删除的小游戏**：拖拽堆叠、点击节奏、拖放布阵、画圈捕捉、点击连消（实现复杂或与其他重复）

### 2.2 模块接口定义（关键接口）

#### EventBus（全局事件总线）
```gdscript
# core/EventBus.gd - 所有模块通过信号通信，降低耦合
extends Node

signal scene_entered(scene_id: String)
signal options_ready(roll_pool: Array)        # 楼主roll池就绪
signal player_action(action_type: String, data: Dictionary)
signal dice_result(roll_value: int, selected_option: Dictionary)
signal minigame_result(success: bool, score: int, keywords: Array)
signal resource_changed(resource_type: String, value: int)
signal forum_event_triggered(event_id: String)
signal node_entered(node_id: int)
signal node_completed(node_id: int, outcome: String)
```

#### MiniGameBase（小游戏基类）
```gdscript
# minigames/MiniGameBase.gd - 所有小游戏继承此类
extends Node2D

@export var duration: float = 15.0  # 小游戏时长
var chaos_pool_ref: Array = []      # 混沌池关键词引用

func _ready():
    start_game()

func start_game():
    # 子类实现：初始化
    pass

func end_game(success: bool, score: int, keywords: Array):
    EventBus.minigame_result.emit(success, score, keywords)
    queue_free()
```

#### NodeScript（节点脚本基类）
```gdscript
# game/nodes/NodeBase.gd - 所有节点继承此类
extends Node2D

@export var node_id: int
@export var preset_options: Array[Dictionary] = []  # 预设选项

func enter_node():
    EventBus.node_entered.emit(node_id)
    # 加载场景资源、初始化解谜玩法

func complete_node(outcome: String):
    EventBus.node_completed.emit(node_id, outcome)
```

#### ResourceManager（资源条）
```gdscript
# core/ResourceManager.gd
extends Node

var qi_ren: int = 0    # 气人值 0-100
var ren_qi: int = 0    # 人气值 0-100

func add_qi_ren(amount: int)
func add_ren_qi(amount: int)
func spend_ren_qi(amount: int) -> bool  # 消耗人气值干涉骰子
func get_qi_ren_level() -> int          # 返回0-3（低/中/高/满）
func get_ren_qi_level() -> int
```

### 2.3 瀑布式开发阶段

#### 阶段1：基础框架（0-12小时）
**目标**：搭好骨架，所有人能在此基础上并行开发

| 任务 | 负责人 | 产出 |
|------|--------|------|
| 项目结构+全局单例 | 程序A | core/ 全部文件 |
| EventBus信号定义 | 程序A | 所有信号签名 |
| 主界面UI骨架 | 程序B | 左右双栏+资源条占位 |
| 数据配置格式定义 | 程序A | data/*.json 模板 |
| MiniGameBase基类 | 程序C | 可继承的基类 |

**关键**：阶段1完成后，所有人可以并行开发各自模块，通过EventBus通信。

#### 阶段2：核心系统（12-28小时）
**目标**：七步循环跑通，资源条生效，选项池工作

| 任务 | 负责人 | 依赖 |
|------|--------|------|
| GameLoop状态机 | 程序A | 阶段1 |
| ResourceManager | 程序A | 阶段1 |
| ChaosPool | 程序A | 阶段1 |
| ForumUI+PostRenderer | 程序B | 阶段1 |
| OptionPool | 程序B | 阶段1 |
| PlayerAction | 程序B | 阶段1 |
| DiceAnimator | 程序B | 阶段1 |
| 3个小游戏实现 | 程序C | MiniGameBase |

#### 阶段3：内容实现（28-40小时）
**目标**：5个节点、论坛事件、话题池、混沌池搭配表填充

| 任务 | 负责人 | 依赖 |
|------|--------|------|
| 5个节点脚本 | 程序A+B | 阶段2 |
| EventManager+事件脚本 | 程序B | 阶段2 |
| 话题池系统 | 程序B | 阶段2 |
| 混沌池搭配表配置 | 策划+程序A | 阶段2 |
| 论坛文案池配置 | 策划 | 数据格式 |
| 曲解/强化演出脚本 | 程序C | 阶段2 |

#### 阶段4：整合调试（40-48小时）
**目标**：合并所有模块，调参，修bug

| 任务 | 负责人 |
|------|--------|
| 模块整合+接口联调 | 全员 |
| 流程跑通测试 | 全员 |
| 数值平衡调整 | 策划+程序A |
| Bug修复 | 全员 |
| 打包发布 | 程序A |

### 2.4 接口合并注意事项

1. **命名规范**：所有文件用PascalCase，信号用snake_case，常量用UPPER_SNAKE
2. **资源路径**：所有资源引用用 `res://assets/...` 绝对路径，避免相对路径冲突
3. **信号优先**：模块间通信只用EventBus信号，不直接调用对方方法
4. **数据驱动**：所有文本、选项、配置放 `data/` 目录JSON文件，不硬编码
5. **场景独立**：每个节点和小游戏是独立场景，通过 `change_scene_to` 切换
6. **Git分支**：每人一个分支（feature/forum、feature/minigame等），阶段4统一合并


## 5. 风险与应对

| 风险 | 概率 | 应对 |
