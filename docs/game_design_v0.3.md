# 《安科勇者》游戏策划文档 v0.3

> **Game Jam 48小时 · 单局1小时 · 纯鼠标 · 半线性**
> 本版聚焦：游戏主要流程循环的可视化与可落地性

---

## 0. 项目定位

| 项目 | 设定 |
|------|------|
| 开发周期 | Game Jam 48小时 |
| 单局时长 | 约60分钟 |
| 操作方式 | 仅鼠标（移动、点击、拖拽、画线） |
| 叙事结构 | 半线性：5个固定主线节点 + 章节间随机填充 |
| 重复游玩 | 单次体验为主，2-3种结局鼓励重玩 |
| 随机性 | 不做全随机/肉鸽，混沌池用**预设搭配表**+少量骰子干扰 |

## 1. 虚构层

### 1.1 世界观

玩家扮演异世界勇者。冒险第一天，脑内突然弹出一个**只有自己能看见**的悬浮界面——长得酷似NGA论坛的安科帖。

- 帖子里的「楼主」是命运本身，用投骰子决定勇者的人生
- 帖子里的「网友」是异界意识，会吐槽、拱火、出馊主意
- 勇者（玩家）发现自己可以回帖，回复会影响接下来的骰子和剧情

### 1.2 玩家回帖机制

论坛左下角会不定期弹出回帖选项。**选项内容大多是勇者自己的吐槽、内心戏、或想对楼主/网友说的话**——不是策略选择，是性格表达。

示例选项（节点③渡河场景）：
```
骰子娘: D100=12 大失败！岸边只有一只小鸡。

○ A. "……楼主你认真的？"
○ B. "小鸡？行吧，反正我现在也没什么尊严了"
○ C. "等等，我先观察一下这只鸡"
```

- 每个选项都对应**明确的游戏效果**（影响骰子权重/注入混沌池关键词/改变网友态度），但玩家**不会被明确告知效果**
- 选项的语气会影响后续网友的回复风格（选了B拱火，网友会更嚣张；选了A冷静，网友会更正经）
- 3秒内不选默认第一项，保持节奏不拖

## 3. 章节与节点框架

整局1小时被切成**5个章节**，每章一个**固定的主线解谜节点**（场景互动玩法，由策划自定义脚本，**不在此文档展开**），中间用随机小游戏和论坛事件填充。

```
章节1 开端  →  章节2  →  章节3  →  章节4  →  章节5 终局
   ①           ②         ③         ④         ⑤
[装备选择]  [连线解谜] [场景解谜] [拼点卡牌] [躲避+回帖]
```

| 节点 | 类型 | 时长 | 玩法骨架 |
|------|------|------|---------|
| ① | 教学关 | 2-3分钟 | 拖拽选择，初始关键词进池 |
| ② | 连线解谜 | 2-3分钟 | 画线连接，符文图案随混沌池变化 |
| ③ | 场景解谜（招牌） | 4-5分钟 | 多解分支，玩家操作催化关键词组合 |
| ④ | 拼点卡牌 | 3-4分钟 | 拖拽卡牌，混沌池关键词作为卡牌消耗 |
| ⑤ | 终局决战 | 4-5分钟 | 鼠标躲避+实时回帖，决定结局 |

**节点的具体剧情、楼层文本、网友回复、分支结构由策划自定义脚本填写。**
此文档只提供框架与流程，不预设具体故事内容。

## 4. 核心循环流程（重点）

### 4.1 循环概览

整个游戏由「论坛↔小游戏」的双向影响循环驱动。**每轮循环约1-3分钟**，一局游戏大约循环20-30次。

### 4.2 循环六步

```
左侧论坛                              右侧游戏
─────────                            ─────────
① 楼主/骰子娘更新  ──关键词──▶  ② 触发小游戏
   投骰子 · 推进剧情                    从池抽取 · 类型由骰子/关键词决定
   网友吐槽                                    │
        ▲                                      ▼
        │                                 ③ 玩家用鼠标游玩
        │                                    操作 · 选择 · 拖拽 · 画线
        │                                      │
        │                                      ▼
        │                                 ④ 产出结果 + 新关键词
        │                                    胜负 · 操作记录 · 触发组合
        │                                      │
   ⑤ 论坛自动续写  ◀──行为反馈──            │
   楼主描述结果 · 网友实时吐槽                  │
        │                                      │
        ▼                                      │
   ⑥ 弹出选项回帖                              │
   3个吐槽/内心戏选项 · 3秒倒计时               │
        │                                      │
        └──选项影响下一轮骰子/事件权重─────────┘
```

### 4.3 双向影响的关键设计

**论坛→游戏**（向右流）：
- 骰子结果决定小游戏**类型**（D100高=战斗类，低=解谜类，极值=特殊事件）
- 网友提到的名词会**高亮**并进入混沌池，作为小游戏的视觉素材
- 论坛事件（骂战、色图、吞楼）会**临时改变游戏层状态**

**游戏→论坛**（向左流）：
- 玩家在小游戏里的胜负、操作记录，会被楼主**自动续写**成剧情
- 玩家在场景里交互的元素，会成为新楼层里的关键词
- 玩家在游戏里触发的混沌池组合，会作为「神展开」楼层出现

**选项回帖→下一轮**（回流）：
- 选项不直接改变当前小游戏，而是**调整下一轮骰子和事件的权重**
- 比如"拱火"选项→下一轮骂战事件概率+30%
- "冷静"选项→下一轮大失败概率-20%
- 这避免了"选项=立刻改变游戏"的机械感，让影响是**缓释**的

### 4.4 与场景解谜节点的关系

5个主线节点是**固定脚本**，不参与上述循环——它们有自己预设的剧情、玩法、分支。

但节点会与循环系统**接口**：
- **进入节点前**：循环正常进行，关键词持续积累到混沌池
- **节点进行中**：节点脚本可以读取混沌池的关键词，作为解谜素材或分支条件
- **节点完成后**：节点产出的关键词回到混沌池，循环继续

即：**循环负责"动态填充"，节点负责"主线骨架"**，两者通过混沌池连接。

## 5. 填充层池

章节间填充单元从以下池中随机抽取。

### 5.1 小游戏池（鼠标操作通用，至少做4个）

| 小游戏 | 操作 | 时长 | 与混沌池交互 |
|--------|------|------|------------|
| 躲避弹幕 | 鼠标移动 | 30-60秒 | 弹幕图案来自混沌池关键词 |
| 画线连符文 | 鼠标画线 | 30-60秒 | 符文颜色对应混沌池颜色标签 |
| 拖拽堆叠 | 鼠标拖拽 | 30-60秒 | 堆叠物是混沌池里的道具 |
| 点击节奏 | 鼠标点击 | 20-40秒 | 节奏点位置由关键词决定 |
| 拉霸/老虎机 | 鼠标点击停止 | 20-30秒 | 滚轮图案来自混沌池 |
| 拖放布阵 | 鼠标拖放 | 30-60秒 | 可布阵单位来自混沌池 |
| 画圈捕捉 | 鼠标画圈 | 20-40秒 | 捕捉对象是关键词精灵 |
| 点击连消 | 鼠标点击 | 30-60秒 | 消除块是关键词图标 |

**通用设计**：
- 失败不惩罚，只触发"大失败"骰子，论坛出现搞笑结果
- 成功获得1-2个新关键词进入混沌池
- 所有小游戏共用一套视觉模板，只换核心玩法

### 5.2 论坛事件池

每个事件会**临时改变游戏层**：

| 事件 | 论坛表现 | 游戏层影响 | 时长 |
|------|---------|----------|------|
| 网友骂战 | 两网友对骂刷屏 | 飘屏弹幕遮挡画面，需点击消除 | 30-60秒 |
| 色图警告 | 某楼被标记违规 | 画面马赛克/黑条，魅惑怪吸引鼠标 | 30秒 |
| 吞楼 | 楼层变"[已删除]" | 对应元素变半透明幽灵，可穿墙 | 至下节点 |
| 挖坟 | 旧帖被顶 | 旧小游戏模式短暂复活混合 | 30-60秒 |
| 直播贴 | 楼主说"实时更新" | 游戏不可暂停，节奏加快 | 30-60秒 |
| 水军冲水 | 大量"顶""沙发" | 水怪NPC挡路 | 30秒 |
| 管理员警告 | 红字公告 | 画面和谐化（血变绿、武器变玩具） | 30秒 |
| 骰子暴走 | 连续大成功/失败 | 混乱模式，规则短暂反转 | 30秒 |
| 标题党 | 标题与内容不符 | 规则与预期相反 | 30秒 |
| 楼主请假 | "今天先到这" | 游戏慢动作，背景变暗 | 20秒过渡 |

**关键**：事件可叠加（骂战+吞楼同时发生，画面又乱又看不清），玩家可通过选项回帖干预走向。

## 6. 混沌池系统

### 6.1 简化版机制

不做完全自由组合，用**预设搭配表**：

```
混沌池关键词（最多保留8个，FIFO淘汰）：
[鸡, 河, 大, 剑, 飞, 色, 删, 怒]
```

**搭配表示例**：

| 触发条件 | 结果事件 |
|---------|---------|
| 鸡 + 大 | 鸡变大，可作载具 |
| 鸡 + 飞 | 鸡飞天，触发空中视角 |
| 鸡 + 剑 | 鸡骑士，敌友不明 |
| 河 + 大 | 河变宽，需要新方案 |
| 色 + 删 | 魅惑怪变幽灵怪 |
| 怒 + 大 | 怒气爆发，攻击力提升 |
| 飞 + 剑 | 飞剑，远程攻击 |

**搭配表规模**：30-50条规则，48小时内可控可测。

### 6.2 关键词来源与消耗

- **来源**：论坛骰子结果、网友回复名词、游戏内操作记录、玩家选项回帖
- **消耗**：节点④（密码锁）作为卡牌消耗、节点⑤（决战）召唤消耗3个

## 7. 美术方案

### 7.1 核心思路：左右割裂 + 右侧场景化

**左侧论坛**与**右侧游戏**采用完全不同的美术风格，形成视觉割裂感——这本身强化了"两个世界"的虚构层。

更重要的是：**右侧游戏的美术随场景变化**，这正好契合Game Jam团队里不同美术擅长不同风格的特点——每个人做自己最擅长的部分。

### 7.2 左侧 · 论坛UI风格

- **统一固定风格**：复古论坛UI，白底黑字蓝链接
- 模仿NGA/早期天涯的视觉语言：用户头像、楼层引用、骰子绿色高亮
- 违规/删除内容用红色/灰色/马赛克
- 字体：等宽或衬线，营造"老论坛"质感

### 7.3 右侧 · 场景化美术

| 章节/场景 | 美术风格建议 | 适合的美术同学 |
|----------|------------|--------------|
| 村庄 | 像素风/简约矢量 | 擅长像素的同学 |
| 森林 | 手绘水彩/线稿 | 擅长手绘的同学 |
| 河流 | 剪影/平面构成 | 擅长平面的同学 |
| 城堡 | 暗黑哥特/厚涂 | 擅长厚涂的同学 |
| 决战 | 高对比赛博/霓虹 | 擅长光效的同学 |

**论坛事件对右侧风格的临时扭曲**：
- 骂战：画面变红、噪点叠加
- 色图：马赛克/圣光遮挡
- 吞楼：画面撕裂、信号不良
- 挖坟：风格短暂复古化或反之高清化
- 管理员警告：全画面和谐滤镜

### 7.4 整体氛围

轻松搞笑、荒诞不经。色彩明亮，但会被论坛事件临时扭曲。
音效：论坛侧打字声、消息提示声；游戏侧按场景变；事件触发有"系统警告"声。

## 8. 48小时开发优先级

### MVP（必须做，约30小时）

1. 主界面框架：左论坛+右游戏双栏
2. 论坛系统：楼层滚动、骰子动画、选项回帖UI
3. 5个主线节点：每个的核心玩法（脚本由策划填）
4. 3-4个随机小游戏
5. 5-6个论坛事件
6. 混沌池搭配表：30条核心规则
7. 3种结局判定

### 应该做（约12小时）

- 完整8个小游戏
- 全部10个论坛事件
- 搭配表扩展到50条
- 音效与BGM
- 视觉特效（马赛克、飘屏弹幕、和谐滤镜）

### 可以不做（如有时间）

- 多周目继承
- 成就系统
- 创作模式（玩家自己写安科）

### 技术栈

- **引擎**：Godot 4.7
- **论坛层**：Control + RichTextLabel + ScrollContainer
- **游戏层**：根据小游戏类型用 Node2D 或 Control
- **混沌池**：全局单例，Dictionary存储
- **数据驱动**：论坛文本、选项、搭配表用JSON/CSV配置
