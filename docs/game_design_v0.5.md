## 3. 双资源条：涌现式设计（核心重设计）

### 3.1 设计哲学

**气人值和人气值不是玩家可花费的资源，而是论坛氛围的被动反映。**

玩家不能"花"它们换好处——那样会把安科的随机性变成玩家的策略游戏，失去风味。

它们的设计原则：
1. **被动积累**：玩家无法直接控制涨跌，只能通过小游戏表现和事件中的选择间接影响
2. **涌现效果**：达到阈值后自动改变系统行为（骰子分布、选项权重、事件触发），玩家无法主动调用
3. **正反馈循环**：高气人让局面更混乱→更多大失败→气人更高；高人气让局面更顺→更多大成功→人气更高
4. **互相制衡**：气人高产生的"骰子极端化"也会偶尔产生大成功→给人气→拉低气人，系统有自我调节

### 3.2 气人值（红，0-100）—— 论坛有多"想搞你"

**被动来源**（玩家无法直接控制）：
- 小游戏大失败 → +15
- 骰子大失败（D100≤10）→ +10
- 离谱选项被骰子选中执行（千年杀失败等）→ +8
- 网友骂战期间不干预 → +3/秒

**涌现效果**（自动触发，玩家无法主动调用）：
| 阈值 | 效果 |
|------|------|
| ≥30 | 骰子娘被惹毛 → D100分布极端化（更多大成功和大失败，少普通结果） |
| ≥50 | 网友变喷子 → 4个固定选项里"离谱选项"权重提升 |
| ≥70 | 管理员警告事件必触发 → 画面和谐化 |
| =100 | 封号结局 → 帖子被锁，烂尾 |

### 3.3 人气值（金，0-100）—— 论坛有多"挺你"

**被动来源**：
- 小游戏大成功 → +15
- 骰子大成功（D100≥90）→ +10
- 解决场景解谜 → +12
- 网友拥护事件 → +5

**涌现效果**：
| 阈值 | 效果 |
|------|------|
| ≥30 | 骰子娘心情好 → D100分布偏正向（更多成功结果） |
| ≥50 | 网友变粉丝 → 4个固定选项里"正经选项"权重提升 |
| ≥70 | 加更事件必触发 → 楼主主动加更，送道具/增益 |
| =100 | 成为楼主结局 → 玩家短暂获得改写剧情权力 |

### 3.4 骚选项的涌现（不再由玩家花钱解锁）

第5槽的骚选项**不再由玩家花人气值购买**，而是通过两种机制涌现：

**机制A：玩家选择加入（保留七步③的玩家能动性）**
- 系统从骚选项池**随机抽2个**展示给玩家
- 玩家可以选1个加入第5槽，或选"不加"直接进入投骰
- 玩家有选择权（加哪个/是否加），但池子是随机的——保留了安科不可控的风味

**机制B：氛围涌现（气人/人气高时自动出现）**
- 气人值≥40时，每轮20%概率网友拱火拱出一个"气人型骚选项"（更离谱、更危险）
- 人气值≥40时，每轮20%概率粉丝推荐一个"人气型骚选项"（更巧妙、更强力）
- 这类骚选项**直接占第5槽**，玩家无需选择

两种机制可叠加：玩家可能既看到2个可选骚选项，又遇到1个氛围涌现的骚选项。

### 3.5 正反馈循环与互相制衡

**气人正反馈**：气人高 → 骰子极端 → 大失败多 → 气人更高（趋向封号结局）
**人气正反馈**：人气高 → 骰子正向 → 大成功多 → 人气更高（趋向成为楼主结局）

**互相制衡（关键）**：
- 气人高让骰子极端化，但极端化**也会产生大成功**
- 大成功给人气 → 人气上升 → 拉低气人的相对优势
- 这让玩家不至于一条路走到黑，双修结局（双≥80）变得有挑战但可达成

### 3.6 与原设计的对比

| 维度 | v0.4（旧） | v0.5（新） |
|------|----------|----------|
| 玩家能否花费 | 能（花人气解锁骚选项、花气人触发混乱模式） | 不能 |
| 骚选项获取 | 玩家花20人气值主动购买 | 系统随机抽2个让玩家选，或氛围涌现自动出现 |
| 资源条作用 | 玩家的策略货币 | 论坛氛围的被动反映 |
| 安科风味 | 被削弱（玩家可花钱买控制） | 保留（玩家无法直接控制，只能间接影响） |

## 4. 核心循环：七步法

### 4.1 循环概览

整个游戏由固定七步循环驱动。每个新场景触发一轮，每轮约30-60秒。10分钟大约循环10-15次。每2-3个普通循环后强制进入下一个节点。

### 4.2 七步详解

#### ① 进入场景（右侧）
勇者出现在新场景，面前有一个事件要素（龙/河/商人/宝箱...）。场景元素会基于混沌池的关键词动态生成。

#### ② 出安价选项（左侧）
系统给出4个固定槽位选项，由要素类型决定：
```
面对【一条龙】, 你打算:
○ 1. 打败龙
○ 2. 魅惑龙
○ 3. 绕过龙
○ 4. 千年杀龙
```
4个选项里通常3个正经+1个离谱。

#### ③ 玩家选择是否加骚选项
- 系统从骚选项池随机抽2个展示
- 玩家可选1个加入第5槽，或选"不加"
- 气人/人气≥40时，可能额外涌现1个氛围骚选项自动占槽
- 玩家有选择权，但池子随机——保留安科不可控风味

#### ④ 投骰子决定
- 玩家点击"投骰"
- 骰子娘投D100，按选项权重分配区间
- **权重受气人/人气影响**（涌现式，非玩家主动）：
  - 气人≥50 → 离谱选项权重↑
  - 人气≥50 → 正经选项权重↑
- D100数值效果：90+大成功 / 70-89成功 / 30-69普通 / 10-29失败 / 10-大失败

#### ⑤ 执行选项
- 小游戏（15-20秒）
- 增加场景元素
- 直接进入下一场景
- 节点解谜（仅节点，第⑤步换成复杂解谜，玩家无感）

#### ⑥ 完成选项
- 资源条被动结算：
  - 大成功 → 人气+15
  - 普通 → 人气+3
  - 失败 → 气人+10
  - 大失败 → 气人+15
- 产出的关键词进入混沌池

#### ⑦ 判定特殊事件
基于资源条状态触发论坛事件：
- 气人≥50 → 50%概率触发骂战
- 人气≥50 → 30%概率触发加更
- 气人≥70 → 必触发管理员警告
- 没触发则回到①，进入下一个场景

### 4.3 节点与普通循环的同构

5个固定主线节点**也走七步循环**，区别只在第⑤步：
- 普通循环：第⑤步是15-20秒小游戏
- 节点：第⑤步换成2-3分钟的复杂解谜（拖拽配装/画线连符文/多元素组合/卡牌拼点/长时段躲避）

玩家从循环结构上无法区分节点与普通循环。**每2-3个普通循环后强制进入下一个节点**，保证10分钟内完成5节点+若干普通循环。

### 4.4 节点间的论坛文案控制

- 每两个节点之间，论坛自动出现的随机文案不超过5句
- 5句从文案池抽取（网友吐槽、骰子娘旁白、楼主补刀）
- 超过5句后必须进入下一个节点

## 5. 小游戏池（15-20秒，参考Undertale）

| 小游戏 | 操作 | 时长 | 与混沌池交互 |
|--------|------|------|------------|
| 躲避弹幕 | 鼠标移动 | 15-20秒 | 弹幕图案来自混沌池关键词 |
| 画线连符文 | 鼠标画线 | 15-20秒 | 符文颜色对应混沌池颜色标签 |
| 拖拽堆叠 | 鼠标拖拽 | 15-20秒 | 堆叠物是混沌池里的道具 |
| 点击节奏 | 鼠标点击 | 15-20秒 | 节奏点位置由关键词决定 |
| 拉霸/老虎机 | 鼠标点击停止 | 10-15秒 | 滚轮图案来自混沌池 |
| 拖放布阵 | 鼠标拖放 | 15-20秒 | 可布阵单位来自混沌池 |
| 画圈捕捉 | 鼠标画圈 | 15-20秒 | 捕捉对象是关键词精灵 |
| 点击连消 | 鼠标点击 | 15-20秒 | 消除块是关键词图标 |

**通用设计**：失败不惩罚，只触发大失败骰子（气人+15但可能触发搞笑事件）；成功获得1-2个新关键词进入混沌池。

| 事件 | 论坛表现 | 游戏层影响 | 触发条件 |
|------|---------|----------|---------|
| 网友骂战 | 两网友对骂刷屏 | 飘屏弹幕遮挡画面，需点击消除 | 气人≥50 |
| 色图警告 | 某楼被标记违规 | 画面马赛克/黑条，魅惑怪吸引鼠标 | 随机 |
| 吞楼 | 楼层变"[已删除]" | 对应元素变半透明幽灵，可穿墙 | 随机 |
| 挖坟 | 旧帖被顶 | 旧小游戏模式短暂复活混合 | 随机 |
| 直播贴 | 楼主说"实时更新" | 游戏不可暂停，节奏加快 | 随机 |
| 水军冲水 | 大量"顶""沙发" | 水怪NPC挡路 | 随机 |

| 粉丝送礼 | 网友送礼 | 获得强力道具 | 人气≥70 |
| 封号警告 | 帖子被锁定 | 进入倒计时烂尾路径 | 气人=100 |

## 7. 混沌池系统

预设搭配表，30-50条规则：

| 触发条件 | 结果事件 |
|---------|---------|
| 鸡 + 大 | 鸡变大，可作载具 |
| 鸡 + 飞 | 鸡飞天，触发空中视角 |
| 鸡 + 剑 | 鸡骑士，敌友不明 |
| 河 + 大 | 河变宽，需要新方案 |
| 色 + 删 | 魅惑怪变幽灵怪 |
| 怒 + 大 | 怒气爆发，攻击力提升 |
| 飞 + 剑 | 飞剑，远程攻击 |

关键词最多保留8个（FIFO淘汰）。来源：骰子结果、网友回复、游戏操作、骚选项。消耗：节点④拼点、节点⑤决战召唤。

## 8. 美术方案

### 8.1 左侧 · 论坛UI风格（统一固定）
复古论坛UI，白底黑字蓝链接，模仿NGA/早期天涯。等宽或衬线字体。

### 8.2 右侧 · 场景化美术（随场景变化）

| 章节/场景 | 美术风格建议 | 适合的美术同学 |
|----------|------------|--------------|
| 村庄 | 像素风/简约矢量 | 擅长像素 |
| 森林 | 手绘水彩/线稿 | 擅长手绘 |
| 河流 | 剪影/平面构成 | 擅长平面 |
| 城堡 | 暗黑哥特/厚涂 | 擅长厚涂 |
| 决战 | 高对比赛博/霓虹 | 擅长光效 |

### 8.3 论坛事件对右侧风格的临时扭曲
- 骂战：画面变红、噪点叠加
- 色图：马赛克/圣光遮挡
- 吞楼：画面撕裂、信号不良
- 挖坟：风格短暂复古化或反之高清化
- 管理员警告：全画面和谐滤镜

## 9. 48小时开发优先级

### MVP（必须做，约30小时）
1. 主界面框架：左论坛+右游戏双栏 + 顶部双资源条
2. 论坛系统：楼层滚动、骰子动画、安价选项UI
3. 七步循环状态机
4. 双资源条的涌现式逻辑（被动积累+阈值效果）
5. 骚选项池与随机抽取机制
6. 5个主线节点（脚本由策划填）
7. 3-4个随机小游戏（15-20秒）
8. 5-6个论坛事件
9. 混沌池搭配表：30条核心规则
10. 结局判定（封号/成为楼主/双修）—— 待资源设计确认后细化

### 应该做（约12小时）
- 完整8个小游戏
- 全部12个论坛事件
- 搭配表扩展到50条
- 音效与BGM
- 视觉特效（马赛克、飘屏弹幕、和谐滤镜）

### 技术栈
- **引擎**：Godot 4.7
- **核心系统**：
  - 论坛层：Control + RichTextLabel + ScrollContainer
  - 游戏层：Node2D或Control
  - 混沌池：全局单例，Dictionary存储
  - 七步循环：状态机（FSM）
  - 资源条：全局单例，信号驱动UI更新；阈值效果用观察者模式自动触发
- **数据驱动**：论坛文本、选项、搭配表用JSON/CSV配置
