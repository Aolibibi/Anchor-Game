## 1. 虚构层

异世界勇者脑内弹出一个只有自己能看见的NGA安科帖。楼主是命运，用骰子决定勇者人生；网友是异界意识，会吐槽拱火。勇者可以回帖，回复会影响后续走向。

## 3. 核心循环：七步法

### 3.1 循环概览

整个游戏由固定七步循环驱动。每个新场景触发一轮，每轮约30-60秒。10分钟大约循环10-15次。

**关键**：5个主线节点**也走这七步**，只是第⑤步「执行」被换成更复杂的解谜玩法。玩家从循环结构上无法区分普通循环与节点。

### 3.2 七步详解

| 步骤 | 位置 | 内容 |
|------|------|------|
| ① 进入场景 | 右侧 | 勇者出现在新场景，面前有一个事件要素（龙/河/商人/宝箱...） |
| ② 出安价选项 | 左侧 | 系统给4个固定选项（3正经+1离谱），由要素类型决定 |
| ③ 玩家可加骚选项 | 左侧 | 系统随机抽2个骚选项，玩家选1个加入第5槽或选不加；气人/人气≥40时可能额外涌现1个 |
| ④ 投骰决定 | 左侧 | 骰子娘投D100，按选项权重分配区间（权重受气人/人气影响） |
| ⑤ 执行 | 右侧 | 小游戏15-20秒 / 增加场景元素 / 直接进入下一场景 / **节点解谜**（仅节点） |
| ⑥ 完成结算 | 双侧 | 资源条被动结算（大成功→人气+15，失败→气人+10等）+ 关键词进混沌池 |
| ⑦ 判定特殊事件 | 左侧 | 基于资源条触发论坛事件 → 回到① |

### 3.3 节点与普通循环的同构

5个固定主线节点**也走七步循环**，区别只在第⑤步：
- 普通循环：第⑤步是15-20秒小游戏
- 节点：第⑤步换成2-3分钟的复杂解谜

玩家从循环结构上无法区分。每2-3个普通循环后强制进入下一个节点。

### 3.4 节点间论坛文案控制

每两个节点之间，论坛自动出现的随机文案不超过5句（网友吐槽、骰子娘旁白、楼主补刀）。

## 5. 小游戏玩法池

### 5.1 设计原则

- 单局**15-20秒**，参考Undertale战斗的紧凑感
- 只用鼠标
- 失败不惩罚，只触发大失败骰子
- 成功获得1-2个新关键词进入混沌池
- 所有小游戏共用一套视觉模板，只换核心玩法

### 5.2 小游戏清单

| 小游戏 | 操作 | 时长 | 与混沌池交互 |
|--------|------|------|------------|
| 躲避弹幕 | 鼠标移动 | 15-20秒 | 弹幕图案来自混沌池关键词 |
| 画线连符文 | 鼠标画线 | 15-20秒 | 符文颜色对应混沌池颜色标签 |
| 拖拽堆叠 | 鼠标拖拽 | 15-20秒 | 堆叠物是混沌池里的道具 |
| 点击节奏 | 鼠标点击 | 15-20秒 | 节奏点位置由关键词决定 |
| 拉霸/老虎机 | 鼠标点击停止 | 10-15秒 | 滚轮图案来自混沌池 |
| 拖放布阵 | 鼠标拖放 | 15-20秒 | 可布阵单位来自混沌池 |
| 画圈捕捉 | 鼠标画圈 | 15-20秒 | 捕捉对象是关键词精灵 |
| 点击连消 | 鼠标点击 | 15-20秒 | 消除块是关键词图标 |

### 5.3 曲解/强化对小游戏的扭曲

- **气人高曲解**：弹幕图案变成"爱心""鸡""马赛克"，或勇者武器变成搞笑道具
- **人气高强化**：背景出现粉丝应援弹幕，或勇者获得临时buff

## 7. 双资源条：气人值与人气值

### 7.1 设计哲学

气人值和人气值不是玩家可花费的资源，而是**论坛氛围的被动反映**。玩家无法直接花费，只能通过小游戏表现和事件中的选择间接影响涨跌。

它们的核心作用：**扭曲玩家选择的结果**——曲解或强化，让演出变得更有节目效果。

### 7.2 气人值（红，0-100）—— 曲解效应

**核心**：气人值越高，玩家选的选项越容易被"故意曲解"成更搞笑的版本。曲解**不是变坏，是变得更节目效果**。

**示例**（玩家选"魅惑龙"）：
- 气人低：正常魅惑小游戏
- 气人中：姿势太骚，龙以为你在挑衅 → 变成躲避爱心弹幕
- 气人高：勇者跳了段钢管舞，龙看吐了疯狂吐息 → 弹幕暴增但龙自己眩晕

**关键**：曲解版本可能比原版更有利。玩家不怕气人高，反而期待搞笑演出。

**被动来源**：大失败+15、骰子大失败+10、离谱选项被选中+8、骂战不干预+3/秒、节点内主动引发骂战+20

**涌现效果**：
| 阈值 | 曲解概率 | 其他效果 |
|------|---------|---------|
| ≥30 | 30% | 骰子娘被惹毛，D100分布极端化 |
| ≥50 | 50% | 网友变喷子，离谱选项权重↑ |
| ≥70 | 70% | 管理员警告必触发 |
| =100 | —— | 封号结局 |

### 7.3 人气值（金，0-100）—— 强化效应

**核心**：人气值越高，玩家选的选项越容易被"强化"成更帅气的版本。

**示例**（玩家选"打败龙"）：
- 人气低：正常战斗小游戏
- 人气中：网友集体鼓励，勇者获得"粉丝应援"buff
- 人气高：网友众筹神剑，勇者变身"论坛之子"，多一段QTE大招演出

**被动来源**：大成功+15、骰子大成功+10、解决解谜+12、网友拥护+5、节点内给的好选项被采纳+8

**涌现效果**：
| 阈值 | 强化概率 | 其他效果 |
|------|---------|---------|
| ≥30 | 30% | 骰子娘心情好，D100偏正向 |
| ≥50 | 50% | 网友变粉丝，正经选项权重↑ |
| ≥70 | 70% | 加更事件必触发 |
| =100 | —— | 成为楼主结局 |

### 7.4 叠加与制衡

- **叠加**：气人+人气都高时，选项同时被曲解+强化 → 又搞笑又帅气（双修结局伏笔）
- **互相制衡**：气人高的曲解偶尔产生大成功→给人气→拉低气人；人气高的强化偶尔太顺被网友挑刺→给气人→拉低人气
- **双修结局**：双≥80触发隐藏结局

### 7.5 玩家论坛参与

- **普通循环**：玩家通过安价选项和骚选项参与，无法直接回复论坛
- **节点内**：玩家可主动引发骂战（气人+20，解锁隐藏骚选项）、主动给出选项建议（30%概率被采纳，但选不选中仍由骰子决定）、拱火/安抚（间接影响下轮权重）

## 8. 混沌池系统

预设搭配表，30-50条规则。关键词最多保留8个（FIFO淘汰）。

**搭配表示例**：

| 触发条件 | 结果事件 |
|---------|---------|
| 鸡 + 大 | 鸡变大，可作载具 |
| 鸡 + 飞 | 鸡飞天，触发空中视角 |
| 河 + 大 | 河变宽，需要新方案 |
| 色 + 删 | 魅惑怪变幽灵怪 |
| 怒 + 大 | 怒气爆发，攻击力提升 |

**来源**：骰子结果、网友回复、游戏操作、骚选项、玩家回复
**消耗**：节点④拼点、节点⑤决战召唤

## 9. 美术方案

### 9.1 左侧 · 论坛UI风格（统一固定）
复古论坛UI，白底黑字蓝链接，模仿NGA/早期天涯。

### 9.2 右侧 · 场景化美术（随场景变化）

| 章节/场景 | 美术风格建议 | 适合的美术同学 |
|----------|------------|--------------|
| 村庄 | 像素风/简约矢量 | 擅长像素 |
| 森林 | 手绘水彩/线稿 | 擅长手绘 |
| 河流 | 剪影/平面构成 | 擅长平面 |
| 城堡 | 暗黑哥特/厚涂 | 擅长厚涂 |
| 决战 | 高对比赛博/霓虹 | 擅长光效 |

### 9.3 论坛事件对右侧风格的临时扭曲
- 骂战：画面变红、噪点叠加
- 色图：马赛克/圣光遮挡
- 吞楼：画面撕裂、信号不良
- 管理员警告：全画面和谐滤镜
- 曲解：武器变鸡毛掸子、敌人表情变搞笑、弹幕变爱心
- 强化：勇者有光环、背景有粉丝弹幕、武器有特效

## 10. 48小时开发优先级

### MVP（必须做，约30小时）
1. 主界面框架：左论坛+右游戏双栏 + 顶部双资源条
2. 论坛系统：楼层滚动、骰子动画、安价选项UI
3. 七步循环状态机
4. 双资源条的曲解/强化逻辑
5. 骚选项池与随机抽取机制
6. 5个主线节点（脚本由策划填）
7. 3-4个随机小游戏（15-20秒）
8. 5-6个论坛事件
9. 混沌池搭配表：30条核心规则
10. 结局判定（封号/成为楼主/双修）

### 应该做（约12小时）
- 完整8个小游戏
- 全部12个论坛事件
- 搭配表扩展到50条
- 音效与BGM
- 视觉特效（马赛克、飘屏弹幕、和谐滤镜、曲解演出）

### 技术栈
- **引擎**：Godot 4.7
- **核心系统**：
  - 论坛层：Control + RichTextLabel + ScrollContainer
  - 游戏层：Node2D或Control
  - 混沌池：全局单例，Dictionary存储
  - 七步循环：状态机（FSM）
  - 资源条：全局单例，曲解/强化用策略模式
- **数据驱动**：论坛文本、选项、搭配表用JSON/CSV配置
