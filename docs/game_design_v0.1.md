# 《安科大乱炖》游戏策划文档

## 1. 核心概念

玩家同时扮演两个角色：
- **论坛层**：一个匿名论坛的读者/参与者，观看一场正在进行中的安科创作
- **游戏层**：安科故事中的"主角"，亲身进入被安科骰子决定的世界

**核心乐趣**：论坛的混沌随机性与游戏的操作性产生化学反应，两边互相"污染"，形成无法预料的荒诞喜剧效果。

## 3. 论坛系统设计

### 3.1 楼层内容结构

每层楼包含：
- **楼层号**：1楼（楼主）、2楼+（回复）
- **用户ID**：楼主、匿名用户A~Z、骰子娘、管理员、OP（版主）
- **内容**：文本、骰子结果、图片、链接
- **状态**：正常、被删除（吞楼）、被封禁、高亮（精华）

### 3.2 论坛事件池

论坛事件分为几类：

| 类型 | 示例 | 对游戏的影响 |
|------|------|------------|
| **安科推进** | 楼主投骰子决定剧情 | 直接改变游戏场景/规则 |
| **网友整活** | 回复建议奇葩选项 | 游戏中出现对应道具/敌人 |
| **骰子暴走** | 连续大成功/大失败 | 游戏进入"混乱模式" |
| **骂战** | 有人喷楼主写得烂 | 游戏画面出现弹幕干扰 |
| **色图事件** | 有人发黄图 | 游戏画面被"马赛克"遮挡，或出现魅惑敌人 |
| **吞楼** | 某楼被系统删除 | 游戏中对应元素消失 |
| **锁帖/冲水** | 帖子被管理处理 | 游戏进入倒计时/强制结局 |
| **挖坟** | 很久以前的帖子被顶上来 | 旧游戏模式复活/混合 |
| **直播贴** | 楼主说"正在实时更新" | 游戏变为实时操作，不可暂停 |
| **标题党** | 标题与内容严重不符 | 游戏规则与预期完全相反 |

### 3.3 特殊角色

- **骰子娘**：一个自动化的NPC，负责投骰子并公布结果。她的存在让"随机"成为游戏世界的物理法则。
- **管理员/OP**：当论坛出现违规内容时介入，会导致游戏世界的"规则变动"。
- **楼层精灵**：吞楼后，被删除内容中的关键词会变成游戏里的幽灵道具，只有特定操作能看到。

## 4. 游戏层系统设计

### 4.1 游戏类型转换表

右侧游戏不是固定类型，而是根据论坛内容动态变化。目标是**学习成本极低**，用鼠标或方向键+一个确认键就能玩。

| 论坛触发条件 | 游戏类型 | 玩法说明 | 示例 |
|-------------|---------|---------|------|
| "躲过了所有弓箭" | 弹幕射击 | 鼠标移动躲避，存活N秒 | 箭从四面八方射来 |
| "需要连接符文" | 连线解谜 | 画线连接相同颜色点 | 画线激活魔法阵 |
| "收集材料" | 下落式消除 | 俄罗斯方块变体 | 堆叠材料炼金 |
| "打牌决胜负" | 极简卡牌 | 5张卡，比大小+效果 | 与NPC拼点 |
| "节奏感很强" | 节奏点击 | 音乐note落下时点击 | 随着BGM战斗 |
| "在迷宫里" | 走格子 | 方向键移动，遇敌战斗 | 探索地图 |
| "一回合决胜负" | 老虎机/拉霸 | 停3个滚轮，组合出效果 | 命运之轮 |
| "被一群人围攻" | 塔防/站位 | 拖放单位到指定位置 | 布阵迎敌 |
| "主角在逃跑" | 跑酷 | 空格/点击跳跃躲避 | 自动奔跑，躲避障碍 |
| "在思考对策" | 限时选择 | 多个选项，倒计时选择 | QTE式决策 |
| "召唤了东西" | 弹球/打砖块 | 反弹球击中目标 | 召唤物撞击敌人 |
| "在钓鱼/等待" | 挂机+时机把握 | 等鱼咬钩，按时机点击 | 耐心小游戏 |

### 4.2 核心设计原则

- **每个游戏时长：30秒~3分钟** —— 不是完整关卡，而是"情境片段"
- **操作极简**：尽量只用鼠标或"方向键+确认键"
- **失败不惩罚**：输了也推进剧情，只是论坛里的结果会变成"大失败"版本
- **胜利有奖励**：获得的关键词/道具会进入"背包"，可以影响后续论坛

## 5. 双向互动机制（核心！）

### 5.1 论坛 → 游戏（单向影响的问题）

如果只是"论坛说了什么，游戏就变成什么"，很快会变得可预测且无聊。需要引入：

#### 关键词污染机制

论坛中的某些关键词会被"高亮标记"并进入游戏世界。但玩家在游戏中的操作会让这些词**变异**：

- 论坛说"勇者拿到了一把剑" → 游戏里玩家确实拿到了剑
- 但玩家用剑去砍了鸡 → 论坛下一楼变成"勇者用剑威胁了一只鸡，鸡变大了"
- 玩家继续用变大的鸡堵住了河 → 论坛出现"鸡堵住了河流，勇者过了河"

**核心规则**：游戏内的物理交互会让关键词产生**意想不到的组合**。

### 5.2 游戏 → 论坛（反向影响）

游戏内的行为和收集的物品，会通过以下方式回到论坛：

1. **楼主自动更新**：游戏结束后，楼主会根据玩家在游戏里的行为写一段新内容
2. **网友实时吐槽**：网友会评论游戏里发生的事（比如"这操作太菜了"）
3. **玩家可以回帖**：玩家可以在左下角输入框回复论坛，回复内容可能成为游戏里的新规则

### 5.3 网状互动（避免循环）

关键设计：**不是"一局游戏→一个结果"**，而是——

- 游戏里有一个**共享的"混沌池"**，存放所有出现过但未解决的关键词
- 论坛和游戏同时往混沌池里丢东西
- 混沌池里的关键词会**自主组合**，产生意想不到的事件

举个例子：

```
论坛层                        游戏层                          混沌池
────────────────────────────────────────────────────────────────────────────
楼主：勇者遇到了一条河          玩家操控勇者在河边                [河]
                              河边有一只鸡                      [河, 鸡]
网友A：用鸡堵住河！            （玩家照做了）                    [河, 鸡, 堵]
                              鸡变大了，堵住了河                [河, 鸡, 堵, 大]
楼主：鸡突然变得巨大            玩家过河了                        [河, 鸡, 堵, 大, 过]
网友B：巨大的鸡会飞吗？        玩家点击鸡，鸡开始飞              [河, 鸡, 堵, 大, 过, 飞]
楼主：D100=98 鸡飞走了！        游戏变成弹幕射击（在鸡背上飞）    [河, 鸡, 大, 飞, 弹幕]
                              玩家躲避弹幕，鸡飞到了城堡        [河, 鸡, 大, 飞, 弹幕, 城堡]
网友C：城堡里有色图！            游戏画面被马赛克遮挡              [河, 鸡, 大, 飞, 城堡, 色图]
管理员：删楼！                 论坛某楼消失，游戏里对应城堡消失   [河, 鸡, 大, 飞, 色图]
                              玩家从天上掉下来...               ...
```

**关键**：混沌池里的关键词会自主组合，不需要玩家刻意去"解题"。游戏的核心乐趣是**看着荒诞的因果链自己长出来**。

## 6. 论坛事件与游戏联动详解

### 6.1 骂战事件

**论坛表现**：
- 某网友喷楼主写得烂，或两个网友开始对骂
- 楼层之间出现红色引战线，文字变大变粗

**游戏影响**：
- 游戏画面出现"弹幕评论"飘屏（类似直播弹幕），遮挡视线
- 背景音乐变吵，出现敲击键盘声
- 玩家可以"安抚"或"拱火"：
  - 安抚：点击飘屏弹幕，消除它们，论坛骂战平息
  - 拱火：不处理，骂战升级，游戏画面更混乱，但可能触发隐藏的"混乱事件"

### 6.2 色图/违规事件

**论坛表现**：
- 某楼发了图片，被系统标记为违规（图片变模糊/显示"内容已隐藏"）
- 管理员即将介入

**游戏影响**：
- 游戏画面被"马赛克"或"黑条"遮挡部分区域
- 出现"诱惑型"敌人（比如马赛克怪），靠近会吸引玩家注意力（鼠标被拉向它）
- 如果玩家打败/避开诱惑怪，论坛里的色图事件会被"忽略"；如果玩家"被魅惑"，论坛会多出一个"楼主也发了一张"的帖子
- 管理员介入后，游戏画面被强制"和谐"——比如所有角色变成火柴人，所有血变成绿色，持续一段时间

### 6.3 吞楼事件

**论坛表现**：
- 某楼层突然变成"[此楼已被删除]"或"[内容审核中]"
- 上下文出现断层，逻辑不连贯

**游戏影响**：
- 被删除楼层对应的游戏元素**变成半透明/不可见**，但仍可交互（"幽灵元素"）
- 玩家可以利用幽灵元素穿墙、找到隐藏路径
- 如果连续吞楼太多，游戏世界的"逻辑"出现漏洞——重力失效、时间倒流、角色分裂成多个

### 6.4 锁帖/冲水

**论坛表现**：
- 帖子被置顶"本帖已锁定"
- 或大量无意义回复冲水（"水经验""顶""沙发"）

**游戏影响**：
- 锁帖：游戏进入倒计时，玩家必须在时间结束前完成当前目标，否则强制结束（进入"烂尾结局"）
- 冲水：游戏里出现大量无关NPC（水怪），它们不攻击只挡路，玩家需要清场或挤过去

### 6.5 挖坟事件

**论坛表现**：
- 一个很久以前的帖子被顶上来
- 内容过时，与现代玩家语境脱节

**游戏影响**：
- 旧的游戏模式复活，并与当前模式**混合**
- 例如：当前是弹幕射击，突然挖坟出现一个"俄罗斯方块"帖子 → 游戏变成"在弹幕射击的同时，还要消方块"的双线操作
- 挖坟的帖子越多，游戏模式越混乱，最终可能变成"所有模式同时运行"的混沌状态

### 6.6 直播贴/实时更新

**论坛表现**：
- 楼主说"正在实时更新，请大家稍等"
- 或"我边玩边写"

**游戏影响**：
- 游戏进入**不可暂停**的实时状态
- 论坛的更新与游戏同步进行，玩家看到楼主在"边写边改"
- 如果玩家操作太慢，楼主会写"主角犹豫了很久"，游戏里时间真的变慢

## 7. 混沌池与关键词系统

### 7.1 关键词来源

关键词从以下渠道进入混沌池：
- 论坛中的**名词**（道具、角色、地点、状态）
- 骰子的**数值结果**（大成功=强力效果，大失败=搞笑/负面效果）
- 游戏内的**操作记录**（击败了谁、用了什么道具、走了什么路线）
- 论坛事件的**副作用**（骂战产生的"愤怒"、色图产生的"魅惑"、吞楼产生的"空洞"）

### 7.2 关键词组合规则

混沌池每积累3-5个关键词，会自动触发一次"组合事件"：

| 关键词组合 | 触发事件 | 游戏效果 |
|-----------|---------|---------|
| 鸡 + 变大 + 河 | 鸡堵河 | 新的过河方式，鸡变成载具 |
| 剑 + 鸡 | 杀鸡取剑？ | 鸡变成带剑的鸡骑士，敌友不明 |
| 弹幕 + 色图 | 马赛克弹幕 | 弹幕变成遮挡视线的马赛克块 |
| 大失败 + 城堡 | 城堡塌了 | 游戏场景变成崩塌状态，限时逃生 |
| 河 + 城堡 + 飞 | 飞天城堡 | 场景变成空中城堡，重力反转 |
| 愤怒 + 管理员 | 删帖风暴 | 游戏元素大量消失，进入虚空 |
| 空洞 + 鸡 | 幽灵鸡 | 出现不可见但可骑的幽灵鸡 |
| 魅惑 + 大成功 | 魅惑大成功 | 所有敌人变成友军，但可能叛变 |

### 7.3 玩家干预

玩家可以通过以下方式干预混沌池：
- **在游戏里组合道具**：把两个道具拖到一块，提前触发组合
- **在论坛回复特定内容**：输入关键词可以"引导"混沌池的走向
- **故意制造大失败**：某些最有趣的组合只有大失败才能触发

## 8. 游戏流程设计

### 8.1 一局游戏的结构

不是"开始→进行→结束"的线性结构，而是**发散式**的：

```
开局：论坛出现一个安科主题（如"勇者斗恶龙"）
  ↓
论坛层开始推进剧情，骰子决定走向
  ↓
游戏层根据论坛内容生成第一个游戏片段
  ↓
玩家操作游戏，产生结果和关键词
  ↓
结果回到论坛，网友反应+新骰子+新帖子
  ↓
论坛事件可能发生（骂战、色图、吞楼等）
  ↓
游戏层根据新的混沌池状态继续/变形/转换
  ↓
...持续循环，没有固定的"关卡"...
  ↓
结局触发条件：
  - 安科故事自然到达结局（楼主写的）
  - 帖子被锁/冲水（强制结局）
  - 混沌池过载（游戏世界崩溃，搞笑结局）
  - 玩家主动在某个节点选择"完结撒花"
```

### 8.2 结局类型

| 结局类型 | 触发条件 | 示例 |
|---------|---------|------|
| **正常结局** | 安科剧情自然完成 | 勇者打败恶龙，论坛撒花 |
| **烂尾结局** | 帖子被锁/楼主跑路 | 游戏突然黑屏，显示"作者太监了" |
| **混沌结局** | 混沌池过载 | 游戏世界物理法则崩溃，所有元素混在一起，搞笑收场 |
| **神展开结局** | 特定关键词组合 | 勇者其实是恶龙的儿子，论坛炸了 |
| **404结局** | 吞楼太多导致逻辑崩溃 | 游戏画面变成404页面，主角在404页面里继续冒险 |
| **骂战结局** | 骂战升级到不可收拾 | 论坛变成战场，游戏变成"键盘侠大战" |
| **色图结局** | 色图事件链过长 | 游戏被"和谐"成宝宝巴士，论坛被封 |
| **挖坟结局** | 挖坟太多 | 古今所有游戏模式同时运行，画面爆炸，搞笑结束 |

## 9. 视觉风格建议

### 9.1 论坛层

- 复古论坛风格：白色底、黑色字、蓝色链接
- 可以模仿5ch、nga或早期天涯论坛的UI
- 用户头像用简单的像素图或默认匿名头像
- 骰子结果用绿色高亮
- 违规/删除内容用红色/灰色/马赛克

### 9.2 游戏层

- **与论坛层形成对比**：游戏层可以是精致的像素风或简约手绘风
- **根据论坛内容动态改变风格**：
  - 正常时：正常的RPG风格
  - 骂战时：画面变红，出现噪点
  - 色图事件时：画面被马赛克/圣光遮挡
  - 吞楼时：画面出现"撕裂"效果，像信号不良
  - 挖坟时：画面变成复古8-bit风或反之变成高清现代风

### 9.3 整体氛围

- **轻松搞笑、荒诞不经**，不要严肃
- 色彩明亮，但会根据论坛事件临时扭曲
- 音效：论坛侧有打字声、消息提示声；游戏侧根据类型变化；事件触发时有特殊的"系统警告"声

## 10. 待讨论的关键问题

1. **游戏操作统一性**：右侧游戏类型不断变化，如何保持操作不让玩家混乱？
   - 建议：鼠标为核心，不同游戏类型下鼠标功能变化（移动、点击、拖拽、画线）
   - 或者：键盘方向键+空格为核心，所有类型都适配这个操作

2. **论坛内容是预设还是生成？**
   - 预设：优点是有剧本质量保障，缺点是重复可预测
   - 生成：优点是新鲜感，缺点是可能写出无意义内容
   - 建议：**混合方案**——预设骨架+局部随机生成+玩家输入填充

3. **混沌池组合的逻辑**
   - 是全随机还是有隐藏规则？
   - 玩家能否通过学习掌握规律，还是永远保持惊喜？

4. **游戏时长**
   - 单局目标时长？是15分钟的快餐还是1小时的沉浸？
   - 是否有存档？论坛帖子是否会跨越多次游玩？

5. **安科的核心机制（骰子）**
   - 骰子结果完全随机？还是根据游戏表现加权？
   - 玩家能否看到骰子的"幕后"？

6. **论坛中的"玩家回复"**
   - 是自由输入？还是有限选项选择？
   - 自由输入更有趣，但技术实现复杂；有限选项容易但可能不够混沌

## 11. 可能的游戏模式

### 11.1 故事模式
- 预设几个经典的安科主题（勇者、侦探、恋爱、克苏鲁等）
- 每个主题有骨架剧本，玩家体验一次完整的"安科从开帖到完结"过程
- 适合单人体验，有结局收集要素

### 11.2 无尽模式
- 一个帖子无限进行下去，没有结局
- 混沌池不断累积，游戏越来越混乱
- 目标是"能撑多久"或"能造出多荒诞的组合"
- 适合重复游玩，每次不同

### 11.3 创作模式
- 玩家自己开帖当"楼主"
- 自己写安科，设定选项和骰子规则
- 系统根据玩家的文字自动生成对应的游戏片段
- 可以分享给其他玩家体验
