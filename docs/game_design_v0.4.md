# 《安科勇者》游戏策划文档 v0.4

> **Game Jam 48小时 · 单局10分钟 · 纯鼠标 · 半线性**
> 本版聚焦：固定循环七步法 + 双资源条 + 节点与普通循环同构

---

## 0. 项目定位

| 项目 | 设定 |
|------|------|
| 开发周期 | Game Jam 48小时 |
| 单局时长 | **约10分钟**（只需撑过呈现即可） |
| 操作方式 | 仅鼠标（移动、点击、拖拽、画线） |
| 叙事结构 | 半线性：5个固定主线节点藏在固定循环中，玩家无法察觉 |
| 重复游玩 | 单次体验为主，2-3种结局鼓励重玩 |

**核心设计哲学**：节点与普通循环同构。玩家看到的永远是「进入场景→出安价选项→投骰→执行→下一场景」这套流程，节点只是其中某个循环被换了「更复杂的解谜玩法」而已，玩家无感。

---

## 1. 虚构层

异世界勇者脑内弹出一个只有自己能看见的NGA安科帖。楼主是命运，用骰子决定勇者人生；网友是异界意识，会吐槽拱火。勇者可以回帖，回复会影响后续走向。

## 3. 双资源条设计（核心创新）

### 3.1 设计哲学

气人值和人气值是**两条对立又互补的轨道**：

- **气人值**（红）：玩家"作死"的代价与收益。越气人越混乱，越混乱越有乐子，但越接近崩坏。
- **人气值**（金）：玩家"魅力"的积累。越有趣越被拥护，越拥护越能解锁骚操作。

**关键设计**：两个资源条不是"好/坏"的对立，而是**两种不同的乐趣方向**：
- 气人值高 = 混沌喜剧路线（论坛骂战、管理员警告、神展开）
- 人气值高 = 英雄叙事路线（粉丝拥护、加更奖励、好结局）
- 两者都高 = 隐藏的"双修结局"

### 3.2 气人值（0-100）

**来源**（增加）：
- 选拱火/挑衅选项：+10
- 小游戏大失败：+15
- 在场景里做出离谱操作（用剑砍鸡、推NPC下河）：+5
- 网友骂战期间不安抚：+5/秒

**消耗途径**：
- **触发"混乱模式"**（消耗30点）：下个小游戏规则反转，但收益翻倍
- **解锁"千年杀"等极端选项**（消耗20点）：在4个系统选项外解锁隐藏高伤/搞笑选项
- **超过70**：自动触发管理员警告事件（画面和谐化）
- **达到100**：触发"封号"事件——帖子被锁，进入倒计时烂尾结局路径

### 3.3 人气值（0-100）

**来源**（增加）：
- 选有趣/机智选项：+5
- 小游戏大成功：+10
- 解决场景解谜：+15
- 网友拥护事件触发：+5

**消耗途径**：
- **添加第5个"骚选项"**（消耗20点）：从隐藏池抽取一个比系统选项更有趣/更强力的选项
- **重投骰子**（消耗40点）：对骰子结果不满意可重投一次
- **超过70**：触发"加更"事件——楼主主动加更一段剧情，送道具
- **达到100**：触发"成为楼主"事件——玩家短暂获得改写剧情的权力

### 3.4 双资源条的互动设计

| 状态 | 触发效果 |
|------|---------|
| 气人值≥70 且 人气值≥70 | "神展开"机会：可强制触发一个混沌池组合 |
| 气人值=100 | 封号结局路径（坏结局） |
| 人气值=100 | 成为楼主结局路径（好结局） |
| 两者都≥80 | 隐藏"双修结局"：勇者既是小丑又是英雄 |
| 气人值比人气值高50+ | 论坛风格整体偏骂战/混乱 |
| 人气值比气人值高50+ | 论坛风格整体偏拥护/正经 |

### 3.5 设计灵感总结

这套系统的核心乐趣在于——**玩家始终在"作死"和"求稳"之间做选择**：
- 想看乐子？气人值拉满，触发管理员警告、骂战、神展开
- 想赢？人气值拉满，解锁骚选项、重投骰子、获得加更奖励
- 想要隐藏结局？必须同时维持两条轨道

而且因为气人值可以"主动消耗"换收益（混乱模式、极端选项），玩家不会一味避免气人——**气人本身就是一种资源**。

## 4. 核心循环：七步法（重点）

### 4.1 循环概览

整个游戏由**固定七步循环**驱动。每个新场景触发一轮，每轮约30-60秒。10分钟大约循环10-15次。

**关键**：5个主线节点**也走这七步**，只是第⑤步「执行选项」被换成了更复杂的解谜玩法。玩家从循环结构上无法区分"这是普通循环"还是"这是节点"。

### 4.2 七步详解

#### ① 进入场景（右侧）
勇者出现在新场景，面前有一个**事件要素**：
- 龙、河、商人、宝箱、守卫、悬崖、迷雾...
- 场景元素会基于混沌池的关键词动态生成（如果池里有"鸡"，可能场景里就有一只鸡）

#### ② 出安价选项（左侧）
系统在论坛左下角给出**4个固定槽位选项**：
```
面对【一条龙】, 你打算:
○ 1. 打败龙
○ 2. 魅惑龙
○ 3. 绕过龙
○ 4. 千年杀龙
```
- 选项内容由**要素类型**决定（龙=战斗类选项，河=通过类选项，商人=交易类）
- 4个选项里通常3个正经+1个离谱（参考安科的"万年杀""色诱"传统）

#### ③ 玩家可加选项
- 玩家可以**花20人气值**解锁第5个"骚选项"，从隐藏池抽取
- 骚选项通常更强力或更搞笑（如"和龙拜把子""给龙讲冷笑话"）
- 玩家也可以**不加**，直接进入投骰

#### ④ 投骰子决定
- 玩家点击"投骰"按钮
- 骰子娘投D100，按选项权重分配区间：
  - 默认4个选项各占25%
  - 骚选项被解锁后占15%，其他选项各调整为21%
  - 玩家**无法直接选**哪个选项执行——只能通过解锁骚选项/调整权重间接影响
- D100的高/低数值有不同效果：
  - 90+：大成功，选项效果加倍
  - 70-89：成功
  - 30-69：普通
  - 10-29：失败
  - 10-：大失败，选项反向执行或触发搞笑意外

#### ⑤ 执行选项
根据选中的选项，进入以下之一：
- **小游戏**（15-20秒）：打败龙→弹幕躲避，魅惑龙→节奏点击
- **增加场景元素**：千年杀龙失败→龙生气，场景里多了"愤怒的龙"要素
- **直接进入下一场景**：绕过龙成功→直接走人
- **节点解谜**（仅节点）：被换成了更复杂的解谜玩法，但形式上仍是"执行选项"

#### ⑥ 完成选项
- 通过小游戏 / 失败 / 解谜完成
- 资源条结算：
  - 大成功：人气+10
  - 普通：人气+3
  - 失败：气人+10
  - 大失败：气人+15，但触发搞笑事件可能人气+5
- 产出的关键词进入混沌池

#### ⑦ 判定特殊事件
基于当前资源条状态，可能触发论坛事件：
- 气人值≥50：50%概率触发骂战事件
- 人气值≥50：30%概率触发加更事件
- 气人值≥70：必定触发管理员警告
- 没触发则直接回到①，进入下一个场景

**完成一轮，进入下一个场景。**直到游戏结束。

### 4.3 节点与普通循环的同构

**关键设计**：5个固定主线节点**也走七步循环**，区别只在于第⑤步的"执行选项"被换成了：

| 节点 | 第⑤步特殊执行 |
|------|--------------|
| ① 装备选择 | 拖拽配装解谜（不再是15秒小游戏，而是2-3分钟拖拽） |
| ② 连线 | 画线连符文解谜 |
| ③ 场景解谜 | 多元素拖拽组合解谜 |
| ④ 拼点 | 卡牌拖拽拼点解谜 |
| ⑤ 决战 | 长时段躲避+实时回帖 |

**玩家视角**：依然是"进入场景→出安价选项→投骰→执行"，只是这次执行"时间长一点、玩法复杂一点"。玩家**意识不到**这是"主线节点"，只会觉得"这关比较有挑战"。

### 4.4 节点间的论坛文案控制

- 每两个节点之间，论坛**自动出现的随机文案不超过5句**
- 这5句从文案池抽取，内容包括：网友吐槽、骰子娘旁白、楼主补刀
- 超过5句后，必须进入下一个节点
- 这样保证节奏紧凑，10分钟内完成5节点+若干普通循环

## 5. 小游戏池（15-20秒，参考Undertale）

所有小游戏**只用鼠标**，单局15-20秒，参考Undertale战斗的紧凑感。

| 小游戏 | 操作 | 时长 | 与混沌池交互 |
|--------|------|------|------------|
| 躲避弹幕 | 鼠标移动 | 15-20秒 | 弹幕图案来自混沌池关键词 |
| 画线连符文 | 鼠标画线 | 15-20秒 | 符文颜色对应混沌池颜色标签 |
| 拖拽堆叠 | 鼠标拖拽 | 15-20秒 | 堆叠物是混沌池里的道具 |
| 点击节奏 | 鼠标点击 | 15-20秒 | 节奏点位置由关键词决定 |
| 拉霸/老虎机 | 鼠标点击停止 | 10-15秒 | 滚轮图案来自混沌池 |
| 拖放布阵 | 鼠标拖放 | 15-20秒 | 可布阵单位来自混沌池 |
| 画圈捕捉 | 鼠标画圈 | 15-20秒 | 捕捉对象是关键词精灵 |
| 点击连消 | 鼠标点击 | 15-20秒 | 消除块是关键词图标 |

**通用设计**：
- 失败不惩罚，只触发"大失败"骰子，气人值+15但可能触发搞笑事件
- 成功获得1-2个新关键词进入混沌池
- 所有小游戏共用一套视觉模板，只换核心玩法

## 6. 论坛事件池

每个事件会**临时改变游戏层**，由第⑦步判定触发：

| 事件 | 论坛表现 | 游戏层影响 | 触发条件 |
|------|---------|----------|---------|
| 网友骂战 | 两网友对骂刷屏 | 飘屏弹幕遮挡画面，需点击消除 | 气人≥50 |
| 色图警告 | 某楼被标记违规 | 画面马赛克/黑条，魅惑怪吸引鼠标 | 随机 |
| 吞楼 | 楼层变"[已删除]" | 对应元素变半透明幽灵，可穿墙 | 随机 |
| 挖坟 | 旧帖被顶 | 旧小游戏模式短暂复活混合 | 随机 |
| 直播贴 | 楼主说"实时更新" | 游戏不可暂停，节奏加快 | 随机 |
| 水军冲水 | 大量"顶""沙发" | 水怪NPC挡路 | 随机 |
| 管理员警告 | 红字公告 | 画面和谐化（血变绿、武器变玩具） | 气人≥70 |
| 骰子暴走 | 连续大成功/失败 | 混乱模式，规则短暂反转 | 随机 |
| 标题党 | 标题与内容不符 | 规则与预期相反 | 随机 |
| 加更事件 | 楼主主动加更 | 获得道具/增益 | 人气≥50 |
| 粉丝送礼 | 网友送礼 | 获得强力道具 | 人气≥70 |
| 封号警告 | 帖子被锁定 | 进入倒计时烂尾路径 | 气人=100 |

## 7. 混沌池系统

### 7.1 简化版机制

不做完全自由组合，用**预设搭配表**：

```
混沌池关键词（最多保留8个，FIFO淘汰）：
[鸡, 河, 大, 剑, 飞, 色, 删, 怒]
```

**搭配表示例**（30-50条规则）：

| 触发条件 | 结果事件 |
|---------|---------|
| 鸡 + 大 | 鸡变大，可作载具 |
| 鸡 + 飞 | 鸡飞天，触发空中视角 |
| 鸡 + 剑 | 鸡骑士，敌友不明 |
| 河 + 大 | 河变宽，需要新方案 |
| 色 + 删 | 魅惑怪变幽灵怪 |
| 怒 + 大 | 怒气爆发，攻击力提升 |
| 飞 + 剑 | 飞剑，远程攻击 |

### 7.2 关键词来源与消耗

- **来源**：骰子结果、网友回复名词、游戏内操作记录、玩家添加骚选项
- **消耗**：节点④拼点作为卡牌、节点⑤决战召唤消耗3个

## 8. 美术方案

### 8.1 核心思路：左右割裂 + 右侧场景化

左侧论坛与右侧游戏采用完全不同的美术风格，形成视觉割裂感。右侧游戏的美术**随场景变化**，正好契合Game Jam团队不同美术擅长不同风格的特点。

### 8.2 左侧 · 论坛UI风格

- 统一固定风格：复古论坛UI，白底黑字蓝链接
- 模仿NGA/早期天涯的视觉语言
- 字体：等宽或衬线，营造"老论坛"质感

### 8.3 右侧 · 场景化美术

| 章节/场景 | 美术风格建议 | 适合的美术同学 |
|----------|------------|--------------|
| 村庄 | 像素风/简约矢量 | 擅长像素 |
| 森林 | 手绘水彩/线稿 | 擅长手绘 |
| 河流 | 剪影/平面构成 | 擅长平面 |
| 城堡 | 暗黑哥特/厚涂 | 擅长厚涂 |
| 决战 | 高对比赛博/霓虹 | 擅长光效 |

**论坛事件对右侧风格的临时扭曲**：
- 骂战：画面变红、噪点叠加
- 色图：马赛克/圣光遮挡
- 吞楼：画面撕裂、信号不良
- 挖坟：风格短暂复古化或反之高清化
- 管理员警告：全画面和谐滤镜

### 8.4 整体氛围

轻松搞笑、荒诞不经。色彩明亮，但会被论坛事件临时扭曲。

## 9. 48小时开发优先级

### MVP（必须做，约30小时）

1. 主界面框架：左论坛+右游戏双栏 + 顶部双资源条
2. 论坛系统：楼层滚动、骰子动画、安价选项UI
3. 七步循环状态机
4. 5个主线节点：每个的核心玩法（脚本由策划填）
5. 3-4个随机小游戏（15-20秒）
6. 5-6个论坛事件
7. 混沌池搭配表：30条核心规则
8. 3种结局判定（封号/成为楼主/双修）

### 应该做（约12小时）

- 完整8个小游戏
- 全部12个论坛事件
- 搭配表扩展到50条
- 音效与BGM
- 视觉特效（马赛克、飘屏弹幕、和谐滤镜）

### 技术栈

- **引擎**：Godot 4.7
- **核心系统**：
  - 论坛层：Control + RichTextLabel + ScrollContainer
  - 游戏层：Node2D或Control，根据小游戏类型
  - 混沌池：全局单例，Dictionary存储
  - 七步循环：状态机（FSM）
  - 资源条：全局单例，信号驱动UI更新
- **数据驱动**：所有论坛文本、选项、搭配表用JSON/CSV配置
